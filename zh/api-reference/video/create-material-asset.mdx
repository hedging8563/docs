---
title: "创建素材资产"
openapi: "POST /v1/videos/assets"
description: "将 Seedance 素材资产上传至 AI Sonar 素材组。"
---

将一个可公开访问的源 URL 导入到 Seedance 素材组中。AI Sonar 会存储该素材以供后续使用；请使用返回的素材资产 `id` 进行生成，而不是使用源 URL。

对于 `library_type: "aigc_avatar"`，`group_id` 是可选的；AI Sonar 将使用或创建组织默认的虚拟人素材组。对于 `library_type: "liveness_face"`，`group_id` 是必填项，且必须是 [获取视觉验证结果](/zh/api-reference/video/get-visual-validation-result) 所返回的 AI Sonar 素材组。

| 类型 | 支持的输入 |
|------|-----------------|
| 图像 | `jpeg`, `png`, `webp`, `bmp`, `tiff`, `gif`, `heic`, `heif`；宽高比 `(0.4, 2.5)`；宽度和高度 `(300, 6000)` px；小于 30 MB。 |
| 视频 | `mp4`, `mov`；`480p`, `720p` 或 `1080p`；2-15 秒；宽高比 `[0.4, 2.5]`；宽度和高度 `[300, 6000]` px；总像素在 409600 到 2068676 之间；最大 200 MB；24-60 FPS。 |
| 音频 | `aac`, `wav`, `mp3`；2-15 秒；最大 15 MB。 |

AI Sonar 会校验源 URL、声明或探测到的媒体格式以及文件大小上限。宽高、宽高比、时长、分辨率、总像素和 FPS 采用 best effort 透传，最终由所选视频服务判断。

创建过程是异步的。请轮询 [获取素材资产](/zh/api-reference/video/get-material-asset) 接口，直到 `status` 变为 `ACTIVE`。对于真人素材组，一个组对应一个已验证的人员；如果上传包含多张人脸或人脸与已验证人员不匹配，可能会导致失败。

## 请求示例

```bash
curl https://api.aisonar.dev/v1/videos/assets \
  -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"library_type":"aigc_avatar","group_id":"group-20260720123456-vrt01","asset_url":"https://example.com/reference.png","asset_type":"Image","asset_name":"Front view"}'
```

## 响应示例

```json
{
  "id": "asset-20260720123456-qn7wr",
  "object": "seedance.material_asset",
  "group_id": "group-20260720123456-vrt01",
  "library_type": "aigc_avatar",
  "asset_name": "Front view",
  "asset_type": "Image",
  "status": "PROCESSING",
  "ready": false,
  "error_message": null,
  "metadata": null,
  "created_at": 1782998400000
}
```
