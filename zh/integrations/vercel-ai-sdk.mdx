## 概述

<Note>
**类型**: 框架或平台

**主要路径**: OpenAI 兼容默认值

**支持置信度**: 推荐的集成模式
</Note>

对于 AI Sonar 而言，Vercel AI SDK 中最稳定的默认选择是 **OpenAI 兼容提供程序 (OpenAI-compatible provider)**。

如果您明确需要 **Responses 原生 (Responses-native)** 行为，可以切换到 OpenAI 提供程序并保持相同的 AI Sonar 基础 URL。

请将此页面视为推荐的集成模式，而非保证 Vercel AI SDK 中的每个辅助功能都受 AI Sonar 支持。

## 推荐默认值：OpenAI 兼容提供程序

```bash
npm install ai @ai-sdk/openai-compatible
```

```typescript
import { createOpenAICompatible } from '@ai-sdk/openai-compatible';

export const aisonar = createOpenAICompatible({
  name: 'aisonar',
  apiKey: process.env.API_KEY,
  baseURL: 'https://api.aisonar.dev/v1',
});
```

## 生成文本

```typescript
import { generateText } from 'ai';
import { aisonar } from './aisonar';

const { text } = await generateText({
  model: aisonar.chatModel('gpt-5.4'),
  prompt: 'Explain AI Sonar in one sentence.',
});

console.log(text);
```

## 流式文本

```typescript
import { streamText } from 'ai';
import { aisonar } from './aisonar';

const result = await streamText({
  model: aisonar.chatModel('gpt-5.4'),
  prompt: 'Write a short poem about coding.',
});

for await (const textPart of result.textStream) {
  process.stdout.write(textPart);
}
```

## 工具调用

```typescript
import { generateText, tool } from 'ai';
import { z } from 'zod';
import { aisonar } from './aisonar';

const result = await generateText({
  model: aisonar.chatModel('gpt-5.4'),
  prompt: 'What is the weather in San Francisco?',
  tools: {
    weather: tool({
      description: 'Get weather in a location',
      parameters: z.object({
        location: z.string(),
      }),
      execute: async ({ location }) => ({
        location,
        temperature: 72,
        condition: 'sunny',
      }),
    }),
  },
});

console.log(result.text);
```

## 结构化输出

```typescript
import { generateObject } from 'ai';
import { z } from 'zod';
import { aisonar } from './aisonar';

const { object } = await generateObject({
  model: aisonar.chatModel('gpt-5.4'),
  schema: z.object({
    name: z.string(),
    role: z.string(),
  }),
  prompt: 'Generate a fake developer profile.',
});

console.log(object);
```

## 如果您明确需要 Responses 原生行为

```bash
npm install ai @ai-sdk/openai
```

```typescript
import { createOpenAI } from '@ai-sdk/openai';

export const aisonarResponses = createOpenAI({
  apiKey: process.env.API_KEY,
  baseURL: 'https://api.aisonar.dev/v1',
});
```

```typescript
import { generateText } from 'ai';
import { aisonarResponses } from './aisonar-responses';

const { text } = await generateText({
  model: aisonarResponses('gpt-5.4'),
  prompt: 'Explain AI Sonar in one sentence.',
});
```

<Note>
对于转发式集成，请使用 `@ai-sdk/openai-compatible` 作为安全默认值。仅当您明确需要基于 `/v1/responses` 构建的提供程序路径时，才切换到 `@ai-sdk/openai`。
</Note>

## 最佳实践

<AccordionGroup>
  <Accordion title="优先选择 openai-compatible">
    对于第三方网关和转发后端，`@ai-sdk/openai-compatible` 通常是最稳妥的起点。
  </Accordion>

  <Accordion title="仅在需要时切换到 Responses">
    如果您需要绑定到 `/v1/responses` 的提供程序行为，请谨慎切换提供程序包，而不是在同一个客户端中混合使用两种模式。
  </Accordion>

  <Accordion title="在服务器端保留密钥">
    切勿在客户端代码中暴露您的 AI Sonar API 密钥。请将提供程序设置放在服务器文件或 API 路由中。
  </Accordion>
</AccordionGroup>
