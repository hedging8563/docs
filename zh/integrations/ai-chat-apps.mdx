## 概述

大多数通用 AI 聊天应用通过标准的 OpenAI 兼容聊天路径连接到 AI Sonar。

在实践中，这意味着：
- 使用 `https://api.aisonar.dev/v1` 作为基础 URL（除非应用明确要求输入完整的端点 URL）
- 使用您的 AI Sonar API key 作为 OpenAI 风格的 API key
- 选择应用可以在标准聊天工作流中使用的模型

<Note>
通用聊天应用通常是**面向聊天补全（chat-completions）的**，而不是 Codex Responses WebSocket 客户端。对于需要特定 Responses 行为的 Codex CLI/App、Cursor 或其他编程工具，请使用专门的集成指南。
</Note>

<Note>
**类型**: 聊天应用

**主要路径**: OpenAI 兼容的聊天补全（chat-completions）

**支持置信度**: 支持的路径
</Note>

## 支持的应用

| 应用 | 平台 | 备注 |
|-------------|----------|-------|
| Open WebUI | Web, 自托管 | 流行的自托管选择，支持 OpenAI 兼容连接 |
| LibreChat | Web, 自托管 | 具有自定义 OpenAI 端点的多提供商聊天工作区 |
| LobeChat | Web, 桌面端, 自托管 | 强大的自定义提供商支持 |
| Cherry Studio | Windows, macOS, Linux | 良好的 OpenAI 兼容提供商流程 |
| Chatbox | Windows, macOS, Linux | 简单的 OpenAI 兼容设置 |
| DeepChat | Windows, macOS, Linux | 带有 OpenAI 兼容提供商设置的桌面客户端 |
| Jan | 桌面端, 自托管 | 优先本地运行，支持远程 OpenAI 兼容提供商 |
| TypingMind | Web, 桌面端 | 使用自定义端点样式 |
| NextChat | Web, 自托管 | 轻量级 ChatGPT 风格客户端，支持基础 URL 覆盖 |
| ChatWise | Windows, macOS | 标准自定义提供商模式 |

## 默认连接值

| 字段 | 值 |
|-------|-------|
| Base URL | `https://api.aisonar.dev/v1` |
| API Key | `sk-your-api-key` |
| API 类型 | OpenAI 兼容 |

如果应用要求输入**完整端点 URL** 而不是基础 URL，请使用：
- Chat Completions: `https://api.aisonar.dev/v1/chat/completions`

## 应用设置示例

<Tabs>
  <Tab title="LobeChat">
    **自托管 LobeChat**

    ```bash
    OPENAI_API_KEY=sk-your-api-key
    OPENAI_PROXY_URL=https://api.aisonar.dev/v1
    ```

    **LobeChat Cloud**
    1. 打开 **Settings**（设置） → **AI Service Provider**（AI 服务提供商）
    2. 选择 **OpenAI**
    3. 输入您的 AI Sonar API key
    4. 将自定义端点设置为 `https://api.aisonar.dev/v1`
  </Tab>

  <Tab title="Cherry Studio">
    1. 打开 **Settings**（设置） → **Model Provider**（模型提供商）
    2. 添加一个 **OpenAI-compatible**（OpenAI 兼容）提供商
    3. 输入：
       - **Name**: AI Sonar
       - **API Base URL**: `https://api.aisonar.dev/v1`
       - **API Key**: `sk-your-api-key`
    4. 添加您想要使用的模型
  </Tab>

  <Tab title="Chatbox">
    1. 打开 **Settings**（设置） → **AI Model**（AI 模型）
    2. 选择 **OpenAI API Compatible**
    3. 输入：
       - **API Host**: `https://api.aisonar.dev/v1`
       - **API Key**: `sk-your-api-key`
       - **Model**: `gpt-5.4-mini` 或 `claude-sonnet-5`
  </Tab>

  <Tab title="TypingMind">
    1. 打开 **Settings**（设置）
    2. 选择 **Custom Endpoint**（自定义端点）
    3. 输入：
       - **Endpoint URL**: `https://api.aisonar.dev/v1/chat/completions`
       - **API Key**: `sk-your-api-key`
    4. 配置您想要启用的模型
  </Tab>

  <Tab title="Open WebUI">
    1. 打开 **Admin Settings**（管理员设置） → **Connections**（连接）
    2. 添加一个 **OpenAI-compatible** 连接
    3. 输入：
       - **Base URL**: `https://api.aisonar.dev/v1`
       - **API Key**: `sk-your-api-key`
    4. 保存并选择您想要启用的模型
  </Tab>

  <Tab title="DeepChat">
    1. 打开 **Settings**（设置） -> **Model Provider**（模型提供商）
    2. 添加一个 OpenAI 兼容提供商
    3. 输入：
       - **Base URL**: `https://api.aisonar.dev/v1`
       - **API Key**: `sk-your-api-key`
       - **Model**: `gpt-5.4-mini`
    4. 保存并进行测试聊天
  </Tab>

  <Tab title="Jan">
    1. 打开 **Settings**（设置） -> **Providers**（提供商）
    2. 添加一个远程 OpenAI 兼容提供商
    3. 输入：
       - **Provider name**: AI Sonar
       - **Base URL**: `https://api.aisonar.dev/v1`
       - **API Key**: `sk-your-api-key`
    4. 选择 `gpt-5.4-mini` 或其他 AI Sonar 模型
  </Tab>

  <Tab title="LibreChat">
    在您的 LibreChat 提供商配置中添加一个 OpenAI 兼容端点，并将其指向：

    ```yaml
    customEndpoints:
      - name: AI Sonar
        apiKey: '${OPENAI_API_KEY}'
        baseURL: 'https://api.aisonar.dev/v1'
        models:
          default: ['gpt-5.4-mini', 'gpt-5.4', 'claude-sonnet-5']
          fetch: false
    ```
  </Tab>
</Tabs>

## 更安全的模型默认值

| 类别 | 建议模型 |
|----------|------------------|
| 通用默认 | `gpt-5.4-mini`, `gpt-5.4`, `gpt-5.5` |
| Anthropic | `claude-sonnet-5`, `claude-opus-4-8`, `claude-fable-5` |
| Google | `gemini-3.5-flash`, `gemini-3.1-flash-lite` |
| DeepSeek | `deepseek-v4-flash`, `deepseek-v4-pro` |

## 提示

<AccordionGroup>
  <Accordion title="在应用支持时优先使用基础 URL">
    当您提供 `https://api.aisonar.dev/v1` 作为基础 URL 而不是完整端点 URL 时，大多数应用更容易配置且更稳定。
  </Accordion>

  <Accordion title="优先使用较小的模型">
    日常聊天请先使用 `gpt-5.4-mini` 或 `deepseek-v4-flash`，然后将更繁重的推理任务交给 `gpt-5.4` 或 `claude-opus-4-8`。
  </Accordion>

  <Accordion title="保持编程工具独立">
    通用聊天应用和编程转发通常需要不同的端点行为。请将 Codex/Cursor 风格的工具保留在它们专门的集成页面上。
  </Accordion>
</AccordionGroup>

## 故障排除

<AccordionGroup>
  <Accordion title="连接错误">
    - 验证基础 URL 是否完全为 `https://api.aisonar.dev/v1`
    - 如果应用自动添加路径，请删除末尾的斜杠
    - 如果应用需要完整端点 URL，请使用 `https://api.aisonar.dev/v1/chat/completions`
  </Accordion>

  <Accordion title="401 Unauthorized（未授权）">
    - 仔细检查您的 API key
    - 确认该 key 在 AI Sonar 控制面板中处于激活状态
    - 确保没有多余的空格或换行符
  </Accordion>

  <Accordion title="模型不可用">
    - 验证模型名称是否完全正确
    - 尝试使用广泛支持的默认模型，如 `gpt-5.4-mini`、`deepseek-v4-flash` 或 `claude-sonnet-5`
    - 在 AI Sonar 文档或控制面板中检查当前的可用性
  </Accordion>
</AccordionGroup>
