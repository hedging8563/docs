## 概述

CrewAI 通过在环境变量或 `LLM` 类中设置模型、API key 和 base URL，支持 OpenAI 兼容的 LLM。

<Note>
**类型**: 智能体框架

**主要路径**: OpenAI 兼容的聊天补全 (Chat Completions)

**支持置信度**: 支持自定义端点路径
</Note>

## 环境变量

```python
import os

os.environ["OPENAI_API_KEY"] = "sk-your-api-key"
os.environ["OPENAI_API_BASE"] = "https://api.aisonar.dev/v1"
os.environ["OPENAI_MODEL_NAME"] = "claude-sonnet-5"
```

## LLM 类

```python
from crewai import Agent, LLM

llm = LLM(
    model="claude-sonnet-5",
    base_url="https://api.aisonar.dev/v1",
    api_key="sk-your-api-key",
)

agent = Agent(
    role="Research analyst",
    goal="Summarize technical tradeoffs clearly",
    backstory="You produce concise implementation notes.",
    llm=llm,
)
```

## 模型建议

对于常规任务，建议使用 `gpt-5.4-mini`；对于需要大量规划的智能体，建议使用 `claude-sonnet-5` 或 `gpt-5.4`；对于更快速、低成本的迭代，建议使用 `deepseek-v4-flash`。

## 端点说明

CrewAI 的 OpenAI 兼容路径最适合 Chat Completions 风格的智能体调用。仅当您选择的工具或 SDK 能够支持这些原生协议时，才直接使用 AI Sonar 原生端点。
