## 概述

OpenHands 可以通过其 LiteLLM/OpenAI 兼容的模型配置来使用 AI Sonar，特别是在您需要自行提供模型、API 密钥和 Base URL 的 CLI 和无头工作流中。

<Note>
**类型**: 编码智能体 (Coding Agent)

**主要路径**: OpenAI 兼容模型配置

**支持置信度**: 支持自定义端点路径
</Note>

## 环境

```bash
export API_KEY="sk-your-api-key"
```

## 模型设置

使用：

```text
Model: openai/claude-sonnet-5
API key: sk-your-api-key
Base URL: https://api.aisonar.dev/v1
```

`openai/` 前缀告知基于 LiteLLM 的路径使用 OpenAI 兼容的提供商配置。

## 无头（Headless）设置

如果您正在为 CLI 或无头使用准备本地默认配置，请运行 OpenHands 设置流程并输入：

```bash
make setup-config
```

当系统提示输入模型详细信息时，请使用带有 `openai/` 前缀的 AI Sonar 模型 ID 以及 AI Sonar 的 `/v1` Base URL。

## 模型建议

使用 `claude-sonnet-5` 或 `gpt-5.4` 进行编码智能体规划，使用 `gpt-5.4-mini` 进行更快的常规循环，并在需要强大的替代推理模型时使用 `deepseek-v4-pro`。
