---
title: "Create Real-Person Verification Session"
openapi: "POST /v1/videos/assets/visual-validation-sessions"
description: "Creates a AI Sonar real-person material verification H5 session."
---

Create a short-lived verification session and redirect your user to the returned H5 link. AI Sonar returns only neutral fields: `session_id`, `h5_link`, `callback_url`, `expires_in`, and `language`.

The session and H5 link expire after 120 seconds. The binding token is provided only by the AI Sonar callback redirect. When the H5 flow completes, AI Sonar redirects to your original `callback_url` with these query parameters:

| Parameter | Meaning |
|-----------|---------|
| `session_id` | AI Sonar verification session ID. |
| `verification_token` | Opaque token used to bind the result. |
| `verification_status` | `succeeded` or `failed`. |
| `result_code` | Result code for your application logic. |

If `verification_status` is `succeeded`, call [Bind Real-Person Verification Result](/api-reference/video/bind-real-person-verification-result). The [test console](https://console.aisonar.dev/seedance-assets) can be used to verify callback handling before you wire the flow into your product.

## Example Request

```bash
curl https://api.aisonar.dev/v1/videos/assets/visual-validation-sessions \
  -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"callback_url":"https://yourapp.example.com/seedance/callback","language":"en"}'
```

## Example Response

```json
{
  "object": "seedance.visual_validation_session",
  "session_id": "svv_abc123",
  "h5_link": "https://verify.example.com/h5?lng=en",
  "callback_url": "https://yourapp.example.com/seedance/callback",
  "expires_in": 120,
  "language": "en"
}
```
